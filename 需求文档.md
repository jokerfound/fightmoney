# 网页版2D搜打撤游戏需求分析文档

## 1. 项目概述
### 1.1 项目背景
本项目旨在开发一个网页版的2D"搜打撤"类型游戏，核心玩法围绕**物资搜刮-战斗对抗-安全撤离**的循环。玩家需要在危险环境中搜集有价值的物资，与敌人交战，并成功撤离以保留战利品。游戏强调风险与回报的平衡，具有高度的重玩价值。

### 1.2 项目目标
- **核心目标**：创建紧张刺激的生存掠夺体验，玩家需要在资源搜集与生存风险间做出策略决策
- **技术目标**：基于Web技术实现，支持跨平台运行，确保流畅的游戏性能和响应式控制
- **经济目标**：建立完整的物资-金币经济循环，激励玩家重复游戏

### 1.3 项目范围
- 单机体验，包含完整的搜刮-战斗-撤离循环
- 对局内游戏系统 + 对局外仓库/商店管理系统
- 支持本地数据持久化（玩家仓库、金币、进度）

## 2. 游戏概述
### 2.1 核心玩法循环
```
进入对局 → 搜刮物资 → 战斗对抗 → 成功撤离 → 仓库管理 → 物资交易
     ↓
   死亡 → 物资全失 → 重新开始
```

### 2.2 主题与风格
- **主题**：末日生存/军事冲突主题，设定在废弃都市、军事基地等环境
- **艺术风格**：写实或轻度像素化的2D美术，突出物资辨识度和战术氛围
- **音效设计**：紧张的环境音效、清晰的物资拾取反馈、激烈的战斗音效

## 3. 目标用户
- **核心用户**：16-35岁喜欢生存、策略、roguelike元素的玩家
- **用户特征**：享受风险决策、资源管理、渐进式成长的游戏体验
- **游戏时长**：单局15-30分钟，适合碎片化时间游玩

## 4. 功能需求

### 4.1 核心游戏系统

#### 4.1.1 物资系统
- **物资分类**：
  - 武器类：手枪、步枪、霰弹枪等（不同伤害、射速、准确度）
  - 弹药类：对应各类武器的弹药
  - 护甲类：头盔、防弹衣等（提供伤害减免）
  - 医疗类：急救包、绷带、止痛药等
  - 工艺品：高价值收藏品（古董、珠宝、技术设备等）
  - 资源类：基础 crafting 材料

- **物资属性**：
  - 价值：决定出售价格
  - 重量：影响玩家移动速度和背包容量
  - 稀有度：影响生成概率和价值
  - 耐久度：部分装备有使用限制

#### 4.1.2 战斗系统
- **战斗机制**：
  - 生命值系统：受到攻击减少生命值，归零即死亡
  - 护甲系统：减少受到的伤害，有耐久度
  - 武器系统：不同武器有不同的伤害、射程、后坐力
  - 弹药管理：需要对应类型弹药才能射击

- **敌人AI**：
  - 巡逻行为：在固定路线巡逻
  - 侦查行为：发现异常后前往检查
  - 追击行为：发现玩家后主动追击
  - 战斗行为：寻找掩体、包抄、撤退等

#### 4.1.3 撤离系统
- **撤离点**：地图上固定的撤离位置
- **撤离条件**：到达撤离点并等待倒计时结束
- **随机事件**：撤离点可能失效或需要特殊条件激活

### 4.2 经济系统

#### 4.2.1 仓库管理系统
- **仓库界面**：网格化物品展示
- **物品整理**：自动整理、手动拖拽排序
- **容量限制**：仓库有空间上限，可通过升级扩展

#### 4.2.2 交易系统
- **商店功能**：
  - 物资出售：将搜刮的物资换成金币
  - 装备购买：用金币购买更好的起始装备
  - 价格波动：部分物资价格随时间波动

- **价值体系**：
  - 工艺品：高价值，低实用性
  - 武器装备：中等价值，高实用性  
  - 医疗物资：中等价值，生存必需品
  - 弹药资源：低价值，消耗品

### 4.3 对局准备系统
- **装备选择**：进入对局前从仓库选择携带的装备
- **风险决策**：携带更好装备增加生存几率，但死亡损失更大
- **保险机制**：可选购买保险，死亡后返还部分装备

## 5. 技术需求

### 5.1 技术架构
- **前端技术**：HTML5 Canvas + JavaScript（考虑使用Phaser.js或PixiJS）
- **数据存储**：浏览器本地存储 + 可能的云存档同步
- **物理引擎**：简单的2D碰撞检测和运动物理

### 5.2 性能要求
- **帧率**：目标60FPS，最低30FPS
- **加载时间**：初始加载<5秒，地图切换<3秒
- **内存使用**：优化资源加载，避免内存泄漏

### 5.3 设备兼容性
- **桌面端**：支持主流浏览器（Chrome、Firefox、Safari、Edge）
- **移动端**：支持触控操作，适配不同屏幕尺寸

## 6. 游戏内容设计

### 6.1 地图设计
- **地图类型**：
  - 废弃都市：建筑物密集，搜刮点多
  - 军事基地：高价值物资，强敌聚集
  - 乡村小镇：中等风险，基础物资
  - 地下设施：复杂地形，特殊工艺品

- **地图要素**：
  - 搜刮点：容器、尸体、特殊位置
  - 障碍物：提供掩护和战术选择
  - 危险区域：辐射区、陷阱区等
  - 撤离点：固定或随机位置

### 6.2 进度系统
- **玩家等级**：通过成功撤离获得经验升级
- **解锁内容**：新地图、新武器、仓库扩展等
- **成就系统**：挑战性目标提供额外奖励

## 7. 用户体验设计

### 7.1 界面设计
- **游戏内HUD**：生命值、护甲、弹药、背包空间显示
- **背包管理**：清晰的物品图标和属性显示
- **地图导航**：小地图或指南针系统

### 7.2 控制方案
- **桌面端**：WASD移动，鼠标瞄准射击，快捷键使用物品
- **移动端**：虚拟摇杆移动，触摸瞄准，虚拟按钮交互
